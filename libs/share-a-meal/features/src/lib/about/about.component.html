Functionalities

Here are all the things that this application can do.

User Registration and Authentication

ID: FR-1

Requirement: The application must provide the option for user registration.


Description: Users should be able to create an account to purchase products, leave reviews, and receive recommendations.


Moscow priority: Must have

Product Display and Purchase

ID: FR-2

Requirement: Users should be able to view and purchase products.


Description: Products should be displayed with relevant information and images. Users should be able to add products to their cart and complete the purchase.


Moscow priority: Must have

Store Reviews

ID: FR-3

Requirement: Users should have the ability to leave reviews for a store.


Description: Each user can place one or more reviews about the store where they made a purchase.


Moscow priority: Should have

Product Categories and Recommendations

ID: FR-4

Requirement: Products must be organized into categories, and users should receive recommendations based on their purchases.


Description: For example, if a user buys a smartphone, recommendations for accessories like cases are generated based on the "smartphones" category.


Moscow priority: Should have

Multistore Functionality

ID: FR-5

Requirement: A product can be sold by multiple stores.


Description: The system should support products available from different stores.


Moscow priority: Clould have
